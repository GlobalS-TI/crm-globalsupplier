# ADR-004 — Zod como validación compartida cliente/servidor

**Status:** Accepted  
**Sprint:** 0

## Decision
One Zod schema per entity in lib/validations/. Imported by server actions and client forms alike.

## Reason
DRY — no duplicated validation logic between layers.

## Rejected alternatives
- Separate client/server schemas: diverge over time, create inconsistent UX vs server errors
